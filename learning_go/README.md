# Learning Go

Learning Go is a broad fundamentals area built around chapter-by-chapter notes, examples, and supporting code. It is the most language-focused part of `go-lab`, and it acts as the foundation for the more application-heavy folders elsewhere in the lab.

## Main pieces

- `book_2024.md` for the broader running notes
- `concept_notes.md` for topic-focused explanations
- `code/` for chapter-aligned examples and exercises
- `code/simplewebapp/` for a small application example
- `media/` for supporting images used by the notes

The `code/` area is organized by numbered sections and works as the practical counterpart to the notes.

## What it covers

The material spans the parts of Go that shape everyday programming instincts:

- predeclared types and zero values
- literals, numeric behavior, strings, and runes
- explicit conversions and Go’s type rules
- control flow, functions, and error handling
- collections and basic data-structure usage
- module and package fundamentals
- practical examples that turn language features into working code

The notes are detailed enough to serve as a real study companion, not just a few reminders. They explain both the syntax and the reasoning behind common Go choices.

## Why it matters

This area matters because reliable Go work is mostly built from small habits: understanding zero values, knowing when conversions are required, recognizing where strings and runes differ, and getting comfortable with explicitness in the language. Those details compound.

Learning Go is less about one finished product and more about building the mental model that makes later services, APIs, and architectural material easier to read and extend.

## Working style

Use `concept_notes.md` when you want focused explanations, `book_2024.md` for broader coverage, and `code/` when you want runnable examples that match the ideas in the notes. The small `simplewebapp` example is a useful bridge from language mechanics into actual application structure.
